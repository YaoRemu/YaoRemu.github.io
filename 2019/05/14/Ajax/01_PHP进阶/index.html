总结

  1. 需求
    1. 用户默认访问的是 登陆页面
      1. 登陆改为 index
      2. 首页换个名字
    2. 登陆页面 输入完毕数据之后 点击提交 跳转到了 首页
      1. 修改登陆页面的一些设置
        1. action
        2. method
        3. name
        4. submit
      2. 跳转到了首页之后 在首页的右上角没有注册跟登陆 取而代之的是
        1. xxx 怎么才来
        2. 删除默认的 注册 跟登陆
        3. 获取通过 post提交的数据 输出在 对应的位置即可
  2. 前后台合作
    1. 前端程序员 写好网站的 静态页面 –> 搞定
      1. 用户限制对于界面的要求越来越高
      2. 主要写页面的各自各样的交互效果
    2. 后端程序员
      1. 根据网站的业务逻辑
        1. 业务逻辑 -> 去登录页 没有注册 去注册页 密码输入三次失败 冻结账号 登陆成功之后去首页 获取最新的 信息展示给用户
  3. get还是post
    1. 如果是 像爱旅行 这种 跳转的方式 提交数据 一般是后端程序员来实现
    2. 后端程序员 根据 需求 选择用哪种方式提交
  4. 页面不跳转 提交数据 ajax
    1. 发送请求给服务器 也分 get 跟post
    2. 工作时 因为后端代码 是后台程序员来编写
